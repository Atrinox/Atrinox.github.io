# C 소개

이번 장에서는 C 프로그램의 기본 구조와 핵심 문법을 배웠다.  
이제부터 진짜 "코드"가 등장한다.

---

# 1. C 프로그램의 기본 구조

C 프로그램은 **함수들의 집합**이다.  
그리고 실행은 항상 `main()` 함수에서 시작한다.

```c
#include <stdio.h>

int main(void)
{
    return 0;
}
````

구성 요소를 정리하면:

* `#include <stdio.h>`
  → 표준 입출력 라이브러리 포함 (`printf()` 사용 가능)

* `int main(void)`
  → 프로그램 시작 함수
  → `int`는 반환형
  → `void`는 인자 없음

* `{ }`
  → 함수 본문(블록)

* `return 0;`
  → 운영체제에 정상 종료를 알림

---

# 2. 변수 선언과 대입

C에서는 **변수를 반드시 먼저 선언**해야 한다.

```c
int num;
num = 1;
```

* `int` → 정수형
* `num` → 식별자(변수 이름)
* `=` → 대입 연산자 (오른쪽 값을 왼쪽에 저장)

왜 선언이 필요할까?

1. 메모리 공간 확보
2. 타입 명확화
3. 오타 방지
4. 코드 가독성 향상

---

# 3. printf()와 서식 지정자

출력은 `printf()`로 한다.

```c
printf("My favorite number is %d\n", num);
```

핵심 요소:

* `%d` → 10진 정수 자리표시자
* `\n` → 줄바꿈 문자 (이스케이프 시퀀스)

이스케이프 시퀀스 예시:

| 코드   | 의미    |
| ---- | ----- |
| `\n` | 줄바꿈   |
| `\t` | 탭     |
| `\b` | 백스페이스 |

# 4. 주석

두 가지 스타일이 있다.

```c
/* 여러 줄 가능 */
```

```c
// 한 줄 주석 (C99 이후)
```

주석은 컴파일러가 무시한다.
가독성과 유지보수성을 위해 필수적이다.

---

# 5. 여러 함수 사용하기

C는 여러 함수를 가질 수 있다.

```c
#include <stdio.h>

void butler(void);  // 함수 프로토타입

int main(void)
{
    butler();
    return 0;
}

void butler(void)
{
    printf("You rang, sir?\n");
}
```

중요 포인트:

* **프로토타입 선언 필요**
* 실행 흐름은 코드 위치가 아니라 "호출 위치" 기준
* `void`는 "비어 있음" 의미

---

# 6. 문법 오류 vs 의미 오류

### 1) 문법 오류 (Syntax Error)

* 세미콜론 누락
* 중괄호 실수
* 주석 닫기 누락

→ 컴파일러가 잡아줌

---

### 2) 의미 오류 (Semantic Error)

```c
n3 = n2 * n2;   // 의도는 세제곱인데 실제는 네제곱
```

* 문법은 맞지만 논리가 틀림
* 컴파일은 되지만 결과가 이상함

디버깅 방법:

* 코드 한 줄씩 추적
* 중간중간 `printf()`로 상태 출력
* 디버거 사용

# 7. 프로그램 상태(State)

특정 시점의 **모든 변수 값의 집합**

디버깅은 결국
"의도한 상태 vs 실제 상태" 비교 과정이다.

---

# 8. 키워드와 식별자

키워드는 변수 이름으로 사용할 수 없다.

예:

* `int`
* `return`
* `void`
* `char`
* `if`

또한 `_`로 시작하는 이름은 시스템 예약일 가능성이 높다.

---

# 요약

* C 프로그램은 함수들의 집합
* 실행은 `main()`에서 시작
* 모든 변수는 선언 후 사용
* `printf()`는 서식 지정자를 사용
* 문법 오류는 컴파일러가 잡음
* 의미 오류는 사람이 잡아야 함
* 디버깅은 프로그램 상태 추적
